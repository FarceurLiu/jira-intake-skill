# 適用情境

## 最適合

- QA 測試反饋整理
- PM 把口語需求整理成 Jira
- 設計調整需求標準化
- 協作角色把回報內容轉成可發派任務
- 開發前的派工守門

## 次適合

- 既有 Jira 任務內容清洗
- 對不完整 Jira 任務回寫補件 comment
- 發派前的 priority / assignee 建議
- 完成後移欄到測試前的檢查
- 先判斷跨平台相似任務是不是鏡像任務、共用規格，或共用 root cause
- 提醒新單可能是 duplicate / follow-up，而不是全新問題

## 不適合單靠這個 skill

- 真正的產品決策拍板
- 純工程除錯分析
- 跨多張 Jira 的大型專案切分
- 缺少附件又缺少文字的極度模糊回報
- 需要真實登入、真實操作網站才能確認的問題
- 完整的 board-level 去重、群組化、批量 triage 自動化

## 最佳落點

把它當成：
- 反饋入口的守門員
- Jira 建單前的品質控制
- 可重複使用的任務整理器

不要把它當成：
- 全自動 PM
- 全自動工程主管
- 可以跳過產品澄清的萬能代理

# 使用限制

## 這個 skill 擅長的事

- 整理模糊反饋
- 區分 bug / 需求 / 規格修正
- 找出缺漏資訊
- 產出較乾淨的 Jira 標題與描述
- 建議 priority、assignee 類型、labels

## 這個 skill 不保證的事

- 不保證一定判斷對產品真正意圖
- 不保證影片內容一定能看懂
- 不保證能憑空知道你們公司的責任人分工
- 不保證不同 Jira 專案欄位完全相同
- 不保證直接建單就一定成功，仍受 Jira 權限與欄位設定影響

## 需要人工確認的情況

- 規格本身還沒定
- 反饋內容互相矛盾
- 牽涉多個問題但被寫成一張單
- 涉及正式環境風險、資料錯誤、金流、權限
- assignee 或 priority 需要商業判斷

## 建議使用邊界

- 先用它做 intake，不要一開始就全自動派工
- 團隊規則未定前，assignee 只建議到角色層，不要直接指名真人
- 對外部使用者回報內容，不要完全依賴單一附件做判斷

# 完全不懂技術也能用：手動建單流程

不需要 API、不需要設定任何東西。  
你只需要 Claude（或 Codex），以及一個能開 Jira 的瀏覽器。

## 開始前：先安裝 skill

第一次使用時，先把 repo 放到 agent 的標準 skill 目錄：
- Claude Code：`~/.claude/skills/jira-intake`
- Codex：`${CODEX_HOME:-$HOME/.codex}/skills/jira-intake`

安裝或更新後，請重新開一個新 session，讓 skill 重新載入。

---

## 這個流程能幫你做什麼

你把反饋丟給 Claude 或 Codex，agent 會幫你：

1. 判斷這是 bug、新需求、還是規格寫錯
2. 看資訊夠不夠，不夠的話告訴你還要問什麼
3. 整理成完整的 Jira 草稿（標題、描述、優先度、驗收條件）

你拿到草稿之後，自己貼進 Jira 建單。  
整個過程不需要任何帳號、設定、或技術背景。

---

## 第一步：把反饋丟給 Claude 或 Codex

可以直接說中文，不需要特定格式。以下都可以：

**情境 1：同事用截圖回報問題**
```
幫我用 jira-intake 整理這個問題：

同事說「首頁按鈕點了沒反應，試了幾次都這樣，不知道是什麼版本」
附件有一張截圖
```

**情境 2：自己測試發現 bug**
```
幫我用 jira-intake 整理：

iOS App 的購物車頁，把商品數量改成 0 之後按確認，
畫面沒有任何反應，商品也沒有被刪掉。
預期行為應該是數量變 0 就自動移除商品。
測試環境是 staging，App 版本 2.3.1。
```

**情境 3：PM 說要改一個功能**
```
jira-intake

PM 說：「完成頁要加一個『分享內容』按鈕，點下去可以複製連結」
沒有設計稿，PM 說參考目前的『複製優惠碼』按鈕樣式就好
```

**情境 4：只有一段影片**
```
jira-intake

這是 QA 傳來的錄影，說有 bug，但沒有說明。
[影片附件]
```

---

## 第二步：看 agent 產出的草稿

Claude 或 Codex 會立即產出草稿。以下是幾種典型回覆：

### 情況 1：直接給你草稿（最常見）

```
**[Bug][iOS] 購物車頁數量改為 0 後確認無反應，商品未移除**

問題描述：iOS 購物車頁，將商品數量改為 0 並點確認後，頁面無反應，商品仍在。預期行為是數量為 0 時自動移除商品。
優先序：高
負責人：iOS 工程師
標籤：ios, ui, bug

這樣理解對嗎？確認後我可以直接建單。
```

**你要做的事**：確認草稿正確後回「確認」，或直接告訴 agent 哪裡要改。

---

### 情況 2：資訊不足，先問一個問題

```
目前理解：首頁有個按鈕點了沒反應。

有一件事需要確認：這是 iOS 還是 Android 發生的？
```

**你要做的事**：補充答案，agent 拿到後立即產出草稿。

---

### 情況 3：草稿附注需要確認的事項

```
**[Feature] 完成頁新增「分享內容」按鈕**

問題描述：PM 要求完成頁加入分享內容按鈕，可複製連結。
優先序：中
負責人：PM 確認後指派
標籤：feature, share
完成條件：⚠️ 按鈕位置與分享內容尚待 PM / 設計確認，建議先對齊規格再動工。

這樣理解對嗎？確認後我可以直接建單。
```

**你要做的事**：先去找 PM / 設計確認，再回來告訴 agent 補充資訊。

---

## 第三步：把草稿貼進 Jira

1. 打開 Jira，進入你的專案
2. 點右上角的「**建立**」或「**Create**」按鈕
3. 填寫各欄位：
   - **Summary（標題）**：貼入建議 Jira 標題
   - **Description（描述）**：貼入 Jira 描述全文
   - **Issue Type**：選 Bug 或 Task（依 agent 建議的類型）
   - **Priority**：選 P2 或對應的優先度
   - **Assignee**：依建議的負責人角色選擇
   - **Labels**：填入建議的標籤
4. 確認後按「**建立**」

完成。

## 常見問題

**Q：我不需要每次都說「jira-intake」嗎？**  
A：如果你有安裝這個 skill，說「幫我整理這個 bug / 需求」通常 agent 也會自動判斷。但加上「jira-intake」可以讓它更精準地走這套流程。

**Q：影片要怎麼給 agent？**  
A：直接把影片連結或截圖附上。如果只有影片而沒有說明，agent 會告訴你還需要補什麼文字說明（影片本身不夠）。

**Q：agent 說資訊不夠，但我沒辦法再追問了怎麼辦？**  
A：告訴 agent「這是目前全部的資訊，先幫我產出最佳草稿，缺少的欄位標注 TBD」。agent 會盡力整理，但會在缺少的地方標出需要補充。

**Q：Codex 也能這樣用嗎？**  
A：可以。把相同的文字貼給 Codex，請它用 jira-intake 整理，會拿到一樣的草稿。Codex 和 Claude 一樣不需要任何設定就能做 intake 分析。

如果你想讓 Codex 直接幫你建 Jira，也可以，但需要先具備可寫入的 Jira 路徑：
- Codex 環境已有 Jira MCP
- 或本機已設定這個 repo 的 `scripts + config + API token`

**Q：我想讓 Claude 或 Codex 直接幫我建 Jira，不想手動貼怎麼辦？**  
A：需要你目前使用的環境有可寫入 Jira 的能力，也就是 Jira MCP，或由工程師做一次性的 API 設定。  
完成後你只需要說「幫我建到 Jira」，其他都自動處理。  
詳細設定方式可看 `ONBOARDING.md` 的路徑 B/C。

# 支援模型與適用情境

這個 skill 本質上是規則型 + 判斷型工作流。
只要模型具備基本的文字理解、結構化輸出能力，就能使用。

## 建議模型能力

至少要能做到：
- 讀懂自然語言描述
- 區分實際行為與預期行為
- 找出資訊缺漏
- 依模板輸出結構化結果
- 不在資訊不足時亂補結論

## 較適合的模型類型

### 高階通用模型
適合正式團隊使用：
- Claude Sonnet 類
- GPT-4.1 / GPT-5 類
- Gemini 1.5 Pro / 2.x Pro 類

優點：
- 比較能處理模糊描述
- 比較會做補問
- 比較能辨識標題與描述矛盾

### 中階快速模型
適合大量初篩：
- Gemini Flash 類
- GPT-4.1 mini / 4o mini 類
- 其他中階指令模型

優點：
- 便宜
- 快
- 適合先做初篩，判斷缺少哪些欄位

限制：
- 比較容易過度自信
- 遇到影片描述不足時，容易亂猜
- 對規格語意反轉比較不穩

## 建議用法

### 用高階模型的情況
- 反饋內容很模糊
- 有影片、截圖、多附件
- 需要判斷 bug / 需求 / 規格修正
- 需要直接產出較完整的 Jira 草稿

### 用中階模型的情況
- 先做大量 intake 初篩
- 先判斷缺哪些欄位
- 只做格式整理，不做最終產品判讀

## 不建議的情況

以下情況不要完全依賴小模型：
- 標題和描述衝突
- 規格本身還沒定
- 需要從影片理解細微 UI 問題
- 需要直接自動指派真人
- 涉及正式環境高風險議題

## 實務建議

- intake 初篩可用中階模型
- 最終 `ready to dispatch` 判定建議用較穩的模型
- 若團隊常遇到語意顛倒、需求與 bug 混寫，優先用高階模型
